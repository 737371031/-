步骤 0 · 先改对 API 域名

主站域名优先，国内加速域名不要当默认入口

日常使用以 https://wuzuapi.com 为主。文档里也特别提醒了，国内加速域名虽然能用，但会有访问波动，所以在 CCS 或其他工具里最好把 API 请求地址改回主站。

https://wuzuapi.com
提醒

如果你之前填的是加速域名，先改回主站，再继续后面的导入和测试。

图 1：在 CCS 的 API 请求地址里改成主站域名即可。
步骤 1

先兑换购买码

如果你是刚购买的新用户，先在网站里输入兑换码完成兑换，再继续创建 API Key。

https://wuzuapi.com/redeem
  • 兑换成功后再去 key 页面，避免后面创建时没有可用分组。
  • 如果兑换失败，优先核对购买码是否完整、是否多复制了空格。
图 2-3：先兑换，再进入密钥页面准备创建 key。
步骤 2

创建 API Key 并选对分组

创建 key 时最容易错的是分组选择。这里必须选择你自己的订阅分组，余额组只适合余额方案使用，不能替代订阅分组。

  • 创建一个普通对话 key，用于 CCS 或日常模型调用。
  • 如果后面还要用 GPT Image 2，再额外创建一个生图 key。
  • 创建完成后把 key 复制好，后续导入和手动配置都要用到。
注意

生图 key 和对话 key 不可以共用，建议一开始就分开创建。

图 4-5：选对分组，然后复制好生成出来的 key。
步骤 3

下载 CC-Switch，准备导入配置

如果你要用 CCS，先下载对应系统的安装包。项目里已经放了 Windows 和 macOS 两个安装包，部署后可以直接从网页下载，不必再跳 GitHub。

  • Windows：使用 `.msi` 安装包。
  • macOS：使用 `.dmg` 安装包。
  • 如果你更习惯原始发布页，也可以跳转 GitHub Release 查看。
图 6：文档原始下载入口，网页里会提供本地下载按钮。
步骤 4

在 key 页面使用导入 CCS 功能

网站里的 key 页面支持直接导入 CCS。按照页面提示点击导入，系统会弹出打开提示，确认后启用对应配置即可。

  • 导入后如果没有自动启用，手动把新配置切到启用状态。
  • 如果出现多个配置，优先检查使用的是不是刚刚导入的那个。
图 7-9：导入、确认打开、启用配置。
步骤 5

按需修改模型名并重启 IDE

如果你要接最新指定版本，文档要求把模型名从 gpt-5.4 改成 gpt-5.4-2026-03-05。修改后一定要重启 IDE，这一步很关键。

gpt-5.4-2026-03-05
成本提醒

文档里同时提醒了 GPT 5.5 的成本更高，大约是 5.4 的两倍，切模型前先确认预算。

图 10：打开配置，改模型名，保存后重启 IDE。
步骤 6

先分清两种使用方式：网页生图或 Codex Skill

GPT Image 2 既可以直接跑网页版本，也可以解压成 Codex Skill，通过 $ 命令在 Codex 里调用。两种方式都需要先配置生图专用 key。

再次强调

生图 key 和普通对话 key 必须分开，少了这一步后面几乎一定会出错。

图 11：这一块主要是 GPT Image 2 Skill 和网页版本的接入说明。
步骤 7

解压 zip 到 `.codex/skills` 目录

把下载到的 gpt-image-2-generator-share-20260520-140609.zip 解压到本机用户目录下的 .codex/skills 中。

C:\Users\你的电脑用户名\.codex\skills
  • 解压完成后检查目录层级，不要多一层外壳文件夹。
  • 如果你是多个技能一起装，留意不要覆盖已有内容。
图 12-13：zip 解压位置和目录放置示意。
步骤 8

配置 key，然后在 Codex 里调用

解压完成后，把生图 key 填到对应配置里。之后打开 Codex，使用 $ 命令选择 GPT Image 2 生图技能，就可以开始生成图片。

如果你更喜欢图形界面，也可以直接双击启用网页版本，但同样要先把 key 配好。

图 14：先配 key，再通过 Skill 或网页模式使用。
步骤 9

先在网站中找到 OpenCode 所需密钥内容

OpenCode 接入不是只复制一个 key，而是先找到网站里提供的 OpenCode 配置入口，复制完整 JSON 配置内容，再导入到 CCSwitch 的自定义供应商中。

高频错误

最容易出错的是 JSON 没复制完整，或者把原有内容没删干净就直接粘贴。

图 15-17：找到 OpenCode，并复制完整 JSON 内容。
步骤 10

在 CCSwitch 里添加供应商并改成自定义配置

打开 CCSwitch 后找到 OpenCode，点击添加供应商，选择自定义配置。进入编辑页后，把原来的 CCSwitch 默认内容删掉，再把刚才复制的 JSON 粘贴进去。

  • 不要只追加内容，要替换完整配置。
  • 如果 JSON 粘贴后格式错乱，先回到网站重新复制一遍。
图 18-21：添加供应商、切换自定义配置、替换为完整 JSON。
步骤 11

填写标识、Base URL、Key 和模型

文档里给了一个推荐标识：wuzu-API。完成自定义配置后，再把 Base URL、Key 和模型补齐，模型建议填 GPT-5.4，然后点击添加即可。

标识：wuzu-API | 模型：GPT-5.4
图 22-25：填好标识、Base URL、Key 和模型后，点击添加完成。

注意事项

这几条是最容易出问题的地方，提前看一眼能省不少时间。

为什么推荐主站域名？

文档里已经说明，加速域名会有波动。做工具接入和长期使用时，默认填主站最稳。

为什么我明明有 key 还失败？

先检查是不是选错了分组，或者把生图 key 和对话 key 混用了。

模型改完为什么还是旧结果？

CCS 文档里强调了很多次，保存后要重启 IDE，很多人就卡在这一步。

OpenCode 最容易错在哪？

不是 Base URL，而是 JSON 没复制完整，或者粘贴前没有把原内容删干净。
